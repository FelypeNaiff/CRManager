PROMPT 06 — Integração com PDV
Implemente a integração do CRM TRUPE com o PDV/Balcão.

Regra obrigatória:
O PDV deve estar integrado com clientes, filhos, estoque, carteira de saldo, vendas, financeiro, histórico e CRM.

No PDV, criar campo principal:
- Buscar cliente por WhatsApp.

Ao informar WhatsApp:
- localizar cliente;
- exibir nome;
- exibir filhos;
- exibir saldo disponível;
- exibir última compra;
- exibir observações importantes;
- permitir cadastro rápido se cliente não existir.

No fluxo de venda:
- selecionar cliente;
- selecionar filho da compra, quando aplicável;
- adicionar produtos;
- validar estoque;
- aplicar desconto, se autorizado;
- permitir uso de saldo/crédito;
- calcular valor final;
- finalizar venda;
- baixar estoque;
- registrar financeiro;
- atualizar histórico do cliente;
- atualizar carteira, se usou crédito;
- atualizar dashboard;
- atualizar indicadores do CRM.

Regras para uso de saldo:
- saldo pode ser usado total ou parcialmente;
- valor usado deve gerar movimentação SAIDA no extrato;
- venda deve guardar valor pago com saldo;
- saldo restante deve ser exibido ao finalizar.

Exemplo:
Total da venda: R$ 230,00
Saldo disponível: R$ 85,00
Saldo utilizado: R$ 85,00
Valor a pagar: R$ 145,00

Integrações obrigatórias:
- PDV baixa estoque;
- PDV atualiza financeiro;
- PDV atualiza CRM;
- PDV atualiza histórico;
- PDV atualiza saldo;
- PDV atualiza dashboard;
- PDV alimenta segmentações futuras de WhatsApp.

Critérios de aceite:
- PDV busca cliente por telefone;
- PDV mostra filhos;
- PDV mostra saldo;
- PDV permite usar saldo;
- venda atualiza histórico;
- venda atualiza estoque;

PROMPT 09 — Regra Global de Integração do Projeto
Aplique esta regra global em todo o projeto CRM TRUPE.

Todas as funcionalidades criadas devem obrigatoriamente ser integradas.

Nenhum módulo pode funcionar isolado.

Fluxo principal obrigatório:
Cliente/Responsável
→ Filhos
→ Vendas/PDV
→ Estoque
→ Trocas/Devoluções
→ Carteira/Saldos
→ Financeiro
→ CRM
→ WhatsApp
→ Dashboard
→ Histórico

Toda ação relevante deve:
- atualizar histórico;
- atualizar dashboard;
- respeitar tenant_id;
- registrar usuário responsável;
- registrar data e hora;
- gerar log;
- manter integridade dos dados;
- não duplicar registros;
- preservar relacionamento com cliente.

Exemplos obrigatórios:

Ao cadastrar cliente:
- criar histórico;
- preparar carteira;
- atualizar dashboard;
- permitir vínculo com filhos;
- preparar para WhatsApp.

Ao cadastrar filho:
- vincular ao cliente;
- atualizar dashboard;
- alimentar segmentação;
- registrar histórico.

Ao finalizar venda:
- vincular cliente;
- vincular filho, se informado;
- baixar estoque;
- atualizar financeiro;
- atualizar histórico;
- atualizar CRM;
- atualizar dashboard.

Ao gerar troca/devolução:
- vincular cliente;
- vincular venda original;
- atualizar estoque;
- atualizar carteira;
- gerar extrato;
- atualizar histórico;
- atualizar dashboard.

Ao usar saldo no PDV:
- baixar saldo;
- gerar movimentação no extrato;
- vincular à venda;
- atualizar histórico;
- atualizar dashboard.

Ao criar campanha WhatsApp:
- respeitar aceite de marketing;
- usar segmentação;
- registrar histórico;
- evitar contatos duplicados.

Revise o projeto e garanta que todos os módulos criados sigam essa regra global.
